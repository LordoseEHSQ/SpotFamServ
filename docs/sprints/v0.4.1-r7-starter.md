3. `deploy/pi-deploy.sh`: schwachen Block 5b entfernen (Entrypoint übernimmt R7).

**Pflicht-Verifikation vor dem Tag (Blind-Spot B2 – CI baut das App-Image NICHT):**
`sh -n backend/docker-entrypoint.sh`; lokal `docker-compose build app && docker-compose up -d`;
Break-&-Heal: `docker compose exec -T app chown root:root /data/audio` → `docker compose restart app`
→ `docker compose exec -T -u www-data app sh -c 'touch /data/audio/.w && echo OK && rm /data/audio/.w'`
→ echte Extraktion (legale CC-Quelle, z. B. `https://upload.wikimedia.org/wikipedia/commons/c/c8/Example.ogg`)
→ download/delete. Hinweis: lokal ist `docker compose` v2 nicht installiert → **`docker-compose`** (v1)
nutzen; Symfony braucht eine `.env` (`cp backend/.env.example backend/.env`, vor Commit entfernen, L-020).
